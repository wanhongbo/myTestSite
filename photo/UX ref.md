# PrivaVault UX 参考说明（对齐当前高保真）

用途：为 UI 开发与联调提供统一参考，避免实现偏差。  
适用范围：`prototype/theme-a`、`prototype/theme-b`、`prototype/index.html`、`prototype/playground.html`。  
关联文档：`doc/产品交互方案.md`。

---

## 1. 设计基线

## 1.1 设备与画布

- 目标画布：`375 x 812`（iPhone 逻辑宽高）
- 页面外层为居中容器，模拟真机外观
- 内容布局默认单列，列表与卡片优先垂直流

## 1.2 双主题策略

- Theme A：浅色磨砂、柔和、信任感
- Theme B：暗色高对比、效率感、力量感
- 要求：**交互行为一致**，仅视觉语言不同

---

## 2. 视觉规范（实现口径）

## 2.1 色彩映射

- Theme A 主色：`#3a68ff`
- Theme A 辅助文本：`#64739a`
- Theme A 容器底：半透明白 + 浅蓝背景渐变
- Theme B 主色：`#d0ff3e`
- Theme B 辅助文本：`#92a0c5`
- Theme B 容器底：深灰蓝底 + 强对比描边

实现建议：
- 抽成 token：`color-primary`、`color-text-muted`、`color-surface`、`color-border`
- 禁止在业务组件直接硬编码颜色（原型映射阶段除外）

## 2.2 圆角与描边

- 主容器圆角：A `30-36`，B `30`
- 卡片圆角：`12-18`
- 按钮圆角：`10-14`
- 描边优先 1px（A）/ 2px（B）

## 2.3 阴影与层级

- Theme A：柔和阴影（浮层感）
- Theme B：弱阴影 + 强描边（硬朗感）
- Modal > Toast > 页面内容 > 背景

---

## 3. 字体与排版

## 3.1 字体方案

- Theme A：`Cabinet Grotesk`（标题）+ `Manrope`（正文）
- Theme B：`Syne`（标题）+ `IBM Plex Sans`（正文）

## 3.2 字号层级（建议）

- 大标题：`30-36`
- 页面标题：`22-28`
- 卡片标题：`16-18`
- 正文：`13-14`
- 辅助信息：`11-12`

## 3.3 文案规则

- 按钮文案用动词：Start / Scan / Save / Upgrade
- 隐私承诺固定短句：`On-device. No upload.`
- 避免绝对化承诺（如“100%识别”）

---

## 4. 导航与结构

## 4.1 底部 Tab（已对齐新版）

- 结构：图标 + 文案（垂直排布）
- 激活态：胶囊背景高亮
- 安全区：使用 `env(safe-area-inset-bottom)`
- 点击热区：不小于 `44x44`

## 4.2 页面分区模板

统一优先级：
1) 页面头部（返回 / 切换主题）  
2) 主任务区域（扫描/编辑/结果）  
3) 次级信息（说明、统计、提示）  
4) 固定操作区（底部按钮或 Tab）

---

## 5. 关键交互行为（实现必须一致）

## 5.1 扫描流程

- 首页 CTA -> 扫描页 -> 结果页
- 扫描页必须有：进度、预计剩余时间、查看结果入口
- 可中断任务需保留恢复能力（产品逻辑层）

## 5.2 结果与编辑流程

- 结果列表项可点击进入编辑页
- 编辑页默认展示敏感区域遮罩
- 保存后进入私密相册（或返回结果，取决于产品配置）

## 5.3 付费流程

- 触发点：批量处理、去水印、容量限制
- 统一跳转 paywall
- 支付失败与恢复购买页面必须可达

## 5.4 设置与恢复

- 我的页入口需可达设置页
- 设置页包含：语言、隐私政策、条款、恢复购买

---

## 6. 状态反馈规范

## 6.1 四类状态

- Loading：可感知进度动画
- Empty：空态图 + 下一步操作
- Error：失败原因 + 重试按钮
- Success：确认反馈 + 去向按钮

对应参考页：
- `theme-a/state-showcase.html`
- `theme-b/state-showcase.html`

## 6.2 轻反馈组件

- Toast：短时、非阻塞、自动消失
- Modal：阻塞式确认/提示，需有关闭路径
- 参考：`prototype/playground.html`

---

## 7. 多语言与主题切换要求

## 7.1 i18n 实现要求

- UI 文案全部走资源文件，不硬编码
- 首发语种：`en`、`es`、`pt-BR`、`ja`、`ko`、`zh-Hans`
- 长文案容器可换行，不允许关键按钮文案被截断

## 7.2 主题实现要求

- 主题切换仅影响视觉 token，不影响信息层级和行为
- 暗色主题需保证可读对比度（文本与背景）
- 建议使用 CSS Variables 或设计 token 系统映射到 Flutter Theme

---

## 8. 页面实现优先级（建议）

P0（必须先做）：
- `home`、`scan`、`result`、`editor`、`vault`、`paywall`、`profile`

P1（上线前补齐）：
- `onboarding`、`permission`、`settings`

P2（体验完善）：
- `scan-empty`、`scan-failed`、`payment-failed`、`restore-success`、`state-showcase`

---

## 9. 开发对齐清单（交付前自检）

- 布局是否按 375 宽基线适配并无横向溢出
- 底部 Tab 是否图标+文字、激活态一致、安全区生效
- 主流程是否可完整走通（首页->扫描->结果->编辑->保存）
- 付费失败与恢复购买是否有完整闭环
- Loading/Empty/Error/Success 是否均有明确反馈
- 多语言是否存在截断、错位、超出容器问题
- 深浅主题切换后可读性是否合格

---

## 10. 原型索引建议用法

- 从 `prototype/index.html` 按流程泳道评审
- 用 `prototype/playground.html` 验证主题/语言/反馈组件
- 评审时固定一套主题先走全链路，再切另一套对照
